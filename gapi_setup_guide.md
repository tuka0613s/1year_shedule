# Googleカレンダー連携用キーの取得手順

Googleカレンダーの予定をアプリに連携（表示・追加・編集・削除）するためには、**「APIキー」** と **「OAuth クライアントID」** の **2種類** が必要です。
以下の手順に沿って、順番に作成してください。

> [!IMPORTANT]
> 取得したキーはパスワードと同等です。他人に教えないでください。（本アプリではブラウザ内にのみ安全に保存されます）

---

## 1. Google Cloudプロジェクトの作成
1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、Googleアカウントでログインします。
2. 画面左上の「Google Cloud」ロゴの右隣にあるプロジェクト選択メニューをクリックします。
3. ポップアップの右上にある「**新しいプロジェクト**」をクリックします。
4. プロジェクト名（例: `My Schedule App`）を入力し、「作成」をクリックします。

---

## 2. Google Calendar APIの有効化
1. 左側のハンバーガーメニュー（≡）から「**APIとサービス**」 > 「**有効なAPIとサービス**」を開きます。
2. 画面上部の「**+ APIとサービスの有効化**」をクリックします。
3. 検索バーに「**Google Calendar API**」と入力します。
4. 検索結果から「Google Calendar API」を選択し、「**有効にする**」をクリックします。

---

## 3. OAuth 同意画面の設定（必須の事前準備）
1. 左側のメニューから「**APIとサービス**」 > 「**OAuth 同意画面**」を開きます。
2. User Typeで「**外部 (External)**」を選択し、「作成」をクリックします。
3. 以下の必須項目を入力します：
   - アプリ名（例: `My Schedule App`）
   - ユーザーサポートメール（ご自身のメールアドレスを選択）
   - デベロッパーの連絡先情報（ご自身のメールアドレスを入力）
4. 「保存して次へ」をクリックします。
5. **テストユーザー** の画面で、「**+ ADD USERS**」をクリックし、**ご自身が普段使っているGoogleアカウントのメールアドレス** を追加して「保存して次へ」をクリックします。

---

## 4. 認証情報の作成

左側のメニューから「**APIとサービス**」 > 「**認証情報**」を開きます。
ここで **【A】APIキー** と **【B】OAuthクライアントID** の2つを作成します。

### 【A】APIキーの作成と制限
1. 画面上部の「**+ 認証情報の作成**」 > 「**API キー**」を選択します。
2. 作成されたAPIキーの名前をクリックして編集画面を開きます。
3. **「アプリケーションの制限」** で **「ウェブサイト」** を選択します。
4. 「ウェブサイトの制限」に以下のURLを追加します。
   - `https://tuka0613s.github.io` （GitHub Pages公開時）
   - `http://127.0.0.1:5500` （ローカル開発時）
5. 「保存」をクリックし、キーをコピーしてアプリの設定欄（右クリックで設定モーダルが開きます）に入力します。

### 【B】OAuth クライアントIDの作成
1. 画面上部の「**+ 認証情報の作成**」 > **「OAuth クライアントID」** を選択します。
2. アプリケーションの種類で **「ウェブ アプリケーション」** を選択します。
3. **「承認済みの JavaScript オリジン」** に以下のURLを追加します。
   - `https://tuka0613s.github.io`
   - `http://127.0.0.1:5500`
4. 「作成」をクリックし、クライアントIDをコピーしてアプリに入力します。

---

## 5. アプリでの連携設定
1. アプリ上部の「**📅 Google連携**」ボタンをクリック。
2. 取得したキーを入力して保存すると、Googleのログイン画面が開きます。
3. 手順3で登録したテストユーザーのアカウントでログインし、「すべて許可」を選択してください。
4. アプリ側で予定を追加・編集・削除すると、実際のGoogleカレンダーにも反映されます。
